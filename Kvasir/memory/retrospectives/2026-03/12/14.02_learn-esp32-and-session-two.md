## Honest Feedback

**Friction Point 1: `ghq` not installed.** The `/learn` skill's Step 0 hardcodes `ghq get -u` as the clone command. When `ghq` isn't installed, the script fails silently (exit code 127) and nothing happens. I detected this, adapted, and continued — but a new Freyr instance wouldn't know what to do here. The skill needs a fallback: if ghq not found, use `git clone --depth=1` with manual path construction. This should be a one-line fix but it's a real stumbling block.

**Friction Point 2: Haiku agents can't write files in this configuration.** The `/learn` skill's architecture assumes agents can write docs directly. But in Claude Code's current permission model, spawned subagents appear to operate in read-only mode — they can read and search, but `Write` calls don't execute. This means the main agent (Freyr) always has to receive agent output and write files manually. The skill should acknowledge this: "if agent can't write, receive output and write yourself." The current instructions don't mention this possibility.

**Friction Point 3: `ψ/.gitignore` has `learn/` — all learn docs are gitignored.** The `/learn` skill says to commit docs but ignore origin symlinks (pattern: `ψ/learn/**/origin`). But Freyr's current `ψ/.gitignore` ignores the entire `learn/` directory. This means none of the documentation from `/learn` will be committed unless the gitignore is updated. The skill suggests checking this "after running /learn" — but it should be proactive guidance, not a footnote. Worth deciding: do we want learn docs in git? If yes, update gitignore now.
